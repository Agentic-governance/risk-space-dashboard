Overview

Risk Grid

GET /data/grid_risk.json ~2.8 MB

Returns an array of risk-scored grid cells covering Japan. Each cell contains P(incident), P(escape), expected harm, and subtype breakdowns.

Fields

FieldTypeDescription
latnumberLatitude (WGS84)
lonnumberLongitude (WGS84)
risk_scorenumberP(incident), 0.0 -- 1.0
expected_harmnumberExpected harm = P(incident) x severity x (1 - P(escape))
p_escapenumberEscape probability based on safe haven proximity
safety_rankstring"S" / "A" / "B" / "C" / "D" safety grade
subtypesobjectIncident count by subtype, e.g. {"theft_bicycle": 45}
haven_count_500mintegerNumber of safe havens within 500m
resolutionstring"ultra" (250m) / "high" (500m) / "mid" (1km) / "low" (5km)

Client-Side Query API

For developers who need filtered queries without a backend, use the Risk Query page:

# Query by lat/lon with radius (returns JSON)
/api/risk.html?lat=35.68&lon=139.75&radius=0.01

# Larger radius
/api/risk.html?lat=34.69&lon=135.50&radius=0.05

This loads grid_risk.json client-side and filters cells within the specified radius. The page returns JSON output directly -- useful for testing and lightweight integrations.

JavaScript: Programmatic Risk Query

Fetch the nearest risk cell for any coordinate using client-side JavaScript:

async function getRisk(lat, lon) {
    const grid = await fetch('https://agentic-governance.github.io/risk-space-dashboard/data/grid_risk.json').then(r=>r.json());
    const nearest = grid.reduce((best, cell) => {
        const d = Math.hypot(cell.lat - lat, cell.lon - lon);
        return d < best.d ? {cell, d} : best;
    }, {cell: null, d: Infinity});
    return nearest.cell;
}

// Usage
getRisk(35.68, 139.75).then(cell => {
    console.log('Risk score:', cell.risk_score);
    console.log('Safety rank:', cell.safety_rank);
    console.log('Expected harm:', cell.expected_harm);
});

Webhook Simulation (Polling Pattern)

Note: Since Risk Space is served as static files from GitHub Pages, true push webhooks aren't possible. However, you can achieve equivalent behaviour by polling the realtime endpoint and comparing snapshots.

Recommended polling strategy:

  • GET /data/realtime_slim.json every 5 minutes
  • Compare with previous snapshot (by event id or date+lat+lon) to detect new events
  • Trigger your own webhook / Slack / email / push notification when new events appear
  • Also poll /data/_version.json daily to detect new data releases

Sample Implementation (Node.js)

const BASE = 'https://agentic-governance.github.io/risk-space-dashboard';
let seen = new Set();

async function pollRealtime() {
  const events = await fetch(BASE + '/data/realtime_slim.json').then(r => r.json());
  const fresh = events.filter(e => {
    const key = (e.id || '') + ':' + e.date + ':' + e.lat + ':' + e.lon;
    if (seen.has(key)) return false;
    seen.add(key);
    return true;
  });
  if (fresh.length > 0) {
    // Trigger YOUR webhook here
    await fetch('https://your-webhook.example.com/hook', {
      method: 'POST',
      headers: {'Content-Type': 'application/json'},
      body: JSON.stringify({source: 'risk-space-mcp', new_events: fresh})
    });
  }
}

// First call populates baseline (won't fire), then poll every 5 minutes
pollRealtime().then(() => setInterval(pollRealtime, 5 * 60 * 1000));

Sample Implementation (Browser / Service Worker)

// In a service worker or background tab
const CACHE_KEY = 'risk-space-last-snapshot';

async function checkForNewEvents() {
  const res = await fetch('/data/realtime_slim.json', {cache: 'no-store'});
  const events = await res.json();
  const prev = JSON.parse(localStorage.getItem(CACHE_KEY) || '[]');
  const prevIds = new Set(prev.map(e => e.id + ':' + e.date));
  const fresh = events.filter(e => !prevIds.has(e.id + ':' + e.date));
  if (fresh.length > 0 && 'Notification' in window) {
    new Notification('⚠ Risk Space: ' + fresh.length + ' new incidents', {
      body: fresh[0].subtype + ' near ' + fresh[0].location
    });
  }
  localStorage.setItem(CACHE_KEY, JSON.stringify(events));
}
setInterval(checkForNewEvents, 5 * 60 * 1000);

Rate Limits & Etiquette

  • Don't poll more frequently than once per minute - underlying data only updates every ~30 minutes
  • Use conditional requests (If-Modified-Since) when possible
  • Cache the previous snapshot locally to minimize bandwidth

CORS & Cross-Origin Access

CORS Headers: GitHub Pages serves all static files with permissive CORS headers (Access-Control-Allow-Origin: *), so client-side fetch() from any origin works without a proxy. You can call any /data/*.json endpoint directly from browser JavaScript on your own domain.

This means you can build third-party dashboards, mobile apps, or browser extensions that consume Risk Space data without any backend or authentication.
